# 🚴‍♂️ Eco-Delivery - Optimisation des livraisons (Projet Data & IA)

## 📌 Contexte

Eco-Delivery souhaite devenir leader de la livraison écoresponsable avec une promesse :
**livraison en 30 minutes avec 0g de CO2**.

Problème actuel :
- Des livreurs sous-utilisés
- Des zones saturées
- Des délais trop longs

👉 Objectif : utiliser la Data et l’IA pour optimiser la flotte de livreurs.

---

## 🎯 Objectifs du projet

- Prédire le temps de livraison
- Identifier les zones de saturation
- Réduire les trajets inutiles
- Améliorer la performance globale

---

## 🧠 Approche

Nous avons adopté une démarche **Agile** avec :
- Pair programming (Nene & Jihen)
- Kanban (Teams)
- User stories orientées métier
- Itérations rapides (MVP)

---

## 📊 Dataset

- Source : Kaggle (Food Delivery Dataset)
- Données : commandes, livreurs, temps de livraison
- Période : 2024–2025

---

## 🔍 Étapes du projet

### 1. Exploration des données (EDA)
- Analyse des variables
- Identification des valeurs manquantes
- Détection des anomalies

### 2. Nettoyage des données
- Suppression des doublons
- Gestion des valeurs manquantes
- Correction des types de données
- Suppression des valeurs aberrantes

### 3. Feature Engineering
- Création de nouvelles variables (ex : heure de commande)
- Transformation des variables catégorielles

### 4. Modélisation
- Dashboard Power BI (présentation des résultats)
- Modèle utilisé : Random Forest (baseline)
- Objectif : prédire le temps de livraison

### 5. Évaluation
- Métrique : MAE (Mean Absolute Error)
